# Triggering tests — create-issue

Bench: skill-creator evals. Target: ≥ 90% of "should" queries, none of the "should NOT" queries.
Generic verbs — the "should NOT" list is the critical part.

## Should trigger

- "create an issue for adding CSV export"
- "file a feature request: dark mode"
- "track this idea for later"
- "add an issue for the login bug"
- "make a ticket for each of these three ideas"
- "raise an issue about the flaky test suite"
- "log a feature: export to XLSX"
- "capture this improvement as a backlog item"

## Should NOT trigger

- "close issue 42" (managing an existing issue)
- "comment on issue 42 that it's fixed" (same)
- "list the open issues" (read-only)
- "implement issue 47" (→ implement-issue)
- "let's brainstorm the design" (standalone brainstorm, nothing to file)
- "I have an issue with my build" ("issue" = problem, not a GitHub ticket)

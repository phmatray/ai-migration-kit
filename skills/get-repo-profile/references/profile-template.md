# Repo-profile schema

The shape of `.claude/skills/repo-profile.md`. `get-repo-profile` fills this from detection (Step 3 of
the skill) and writes it; `create-issue` / `implement-issue` / `merge-pr` read it at their preconditions
step. Keep **every section** — leave `<!-- TODO -->` markers for anything undetectable rather than
dropping the field. Values are consumed *verbatim* (label strings, commit identity, commands), so record
them exactly as the repo uses them.

## Worked example — a .NET web app (`.claude/skills/repo-profile.md`)

What detection might produce for a typical .NET repo (fictional `acme/report-hub`), as a reference
for the grain each field wants:

````markdown
# Repo profile

## Identity
- **Repo:** acme/report-hub
- **Default branch:** main

## Commit identity
- **Author line:** `git -c user.email=dev@example.com -c user.name="Jane Dev"`
- **Source:** CLAUDE.md commit-identity rule (use the GitHub identity, not the work email)

## Build & test
- **Build:** `dotnet build`
- **Full test:** `dotnet test` (integration suites need a local Postgres — CI-only here)
- **Single-suite filter:** `dotnet test --filter "FullyQualifiedName~<Suite>"`
- **Format/lint apply:** `dotnet format ReportHub.sln`
- **Format/lint verify (gate):** `dotnet format ReportHub.sln --verify-no-changes --no-restore`
- **Prerequisites / caveats:** integration tests need `docker compose up db` — run unit suites locally
  via the filter; `IDE1006` naming can't be auto-fixed (rename by hand). Front-end tests: `npm test` in `web/`.

## CI gates
- `dotnet build`
- `dotnet test`
- `dotnet format ReportHub.sln --verify-no-changes --no-restore`

## Integration style
- **Merge mode:** squash
- **PR title convention:** Conventional Commits prefix `<type>(scope):` required (semantic-PR-title CI
  check; feeds `release-please`) — derive from issue label (`bug`→`fix`, `enhancement`→`feat`), don't
  reuse the issue title verbatim; title also ends in `(#issue)`; squash appends `(#PR)` →
  `<type>(scope): subject (#issue) (#PR)`
- **Branch naming:** `feat/<issue>-<slug>`

## Labels
- **Type:** `bug`, `enhancement`
- **Priority tiers:** `priority: high` (Tier 1 — core deliverable) · `priority: medium` (Tier 2 — expected, partial/absent) · `priority: low` (Tier 3 — polish)
- **Effort sizes:** `effort: S` (~hours–1d) · `effort: M` (~few days) · `effort: L` (~1–2wk / cross-layer) · `effort: XL` (multi-week/phased)
- **Scope/area:** `area: api` · `area: web` · `area: export` · `area: ci` — exactly one per issue; finer scope via the issue-template Area dropdown

## Issue templates
- **Location:** `.github/ISSUE_TEMPLATE/`
- **Forms:** feature_request.yml, bug_report.yml, config.yml
- **Default for ideas:** feature_request · **for defects:** bug_report

## Conflict hot-spots
| File | Why it collides | Resolution |
|------|-----------------|------------|
| `Directory.Build.props` (`<Version>`) | every feature bumps the patch version | take the **higher** |
| `CHANGELOG.md` | everyone appends | **union** |
| `README.md`, `docs/` | parallel features document themselves | **union** |
| Snapshot files `*.verified.txt` | two branches change output | **regenerate** (re-run the test, accept `.received.txt`) |
| `web/package-lock.json` | lockfile churn | **regenerate** (`npm install`) |
| Test files | both append tests | **union** |
| `.csproj`, `ReportHub.sln` | both add files/projects | **union** item groups |

## Architecture grain / invariants
- Keep `Domain/` free of infrastructure concerns — no EF or HTTP types in domain models.
- Touch layers in order: domain (`Domain/`) → application (`Application/`) → infrastructure (`Infrastructure/`) → API (`Api/`) → tests.
- Do NOT set `TreatWarningsAsErrors` — generated client code may warn and must still build.

## Worktree home
- `.claude/worktrees/` (git-ignored)

## Environment gotchas
- Raw `git fetch`/`push` can be sandbox-blocked while `gh` works (`port 443` timeout) — re-run those with the sandbox disabled; use `git -C <path>`, not `cd`.
- The integration-test DB is CI-only here; run unit suites locally via the single-suite filter.
````
